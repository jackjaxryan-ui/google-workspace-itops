# Onboarding Checklist (Google Workspace + Slack + Trello)

## Inputs needed (from HR/Manager)
- Full name
- Personal email (for initial contact if needed)
- Work email (desired format)
- Department / role
- Start date
- Manager name
- Required access (Shared Drives, Slack channels, Trello boards)

## 1) Google Workspace account
- [ ] Create user in Admin Console
- [ ] Assign correct OU (Org Unit)
- [ ] Set temporary password + require change at first login
- [ ] Add recovery email/phone if policy allows

## 2) Security baseline
- [ ] Enforce 2-Step Verification (MFA)
- [ ] Confirm security keys / authenticator method (per policy)
- [ ] Review sign-in risk settings (if applicable)
- [ ] Ensure user is covered by password policy

## 3) Groups & access
- [ ] Add user to identity/role group(s) (example: media-editors / media-viewers)
- [ ] Confirm Shared Drive access is inherited via permission groups
- [ ] Avoid direct permissions to the user (use Groups only)

## 4) Google Drive / Shared Drives validation
- [ ] User can see required Shared Drives
- [ ] Test create/edit permissions (if contributor)
- [ ] Confirm folder naming conventions and where to store work

## 5) Slack
- [ ] Invite user to workspace
- [ ] Add to required channels (team + announcements + project channels)
- [ ] Apply security settings (MFA/session settings if enforced)

## 6) Trello
- [ ] Add user to required boards
- [ ] Create onboarding card / task list for the new hire
- [ ] Assign owner (manager) + due dates

## 7) Documentation and handover
- [ ] Send welcome email with first-day instructions
- [ ] Document what access was granted (groups, drives, channels, boards)
- [ ] Log completion in ticketing system / tracker
